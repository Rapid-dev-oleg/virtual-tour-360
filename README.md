# 360° Туры — прототип

Демо-приложение для создания виртуальных 3D-туров по недвижимости **со смартфона, без спецоборудования**.
Mobile-first, полностью клиентское (деплой на Vercel как статика).

## Флоу

1. **Мои туры** — список туров + демо-примеры.
2. **Создать тур** — панорамный тур (360°-фото) или видео-проходка.
3. **Редактор** — добавляешь точки (панорамы комнат), тапом ставишь переходы между ними, выбираешь стартовую точку.
4. **Просмотр** — плеер с гироскопом и стрелками-переходами по полу (как Matterport/Яндекс).
5. **Поделиться** — публичная ссылка (Web Share API / копирование).

## Технологии

- **Vite + React** (mobile-first, Tailwind)
- **[Photo Sphere Viewer](https://photo-sphere-viewer.js.org/) v5** + `VirtualTourPlugin` — движок тура (сцены, 3D-стрелки, гироскоп), `MarkersPlugin` — расстановка переходов в редакторе
- Хранение туров — `localStorage` (для демо). Демо-туры зашиты в код и работают по ссылке у любого.

## Запуск

```bash
cd web
npm install
npm run dev      # http://localhost:5173
npm run build    # прод-сборка в web/dist
```

Vercel: Root Directory = `web`, framework preset = Vite (build `vite build`, output `dist`).

## Ограничения демо и что дальше

- **Кросс-девайс шаринг созданных туров** пока не работает (localStorage per-browser). Демо-туры — работают у всех.
  → Следующий шаг: **Vercel Blob** для панорам/видео + метаданных тура.
- **Видео** хранится только в текущей сессии.
  → Задел под **3D-реконструкцию (Gaussian Splatting)**: видео-проходка → серверный GPU-пайплайн → свободное перемещение по сцене.
- Панорамы в демо — плейсхолдеры из открытого набора. Реальные интерьерные 360°-фото подставляются в `web/public/demo/` или снимаются через приложение.
- **Гид по съёмке (guided capture)** на ARKit/ARCore — ключевая фича для неопытных арендодателей — в роадмапе.
