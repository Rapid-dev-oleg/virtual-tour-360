# Пайплайн «видео → 3D-тур» (каркас)

> Статус: **каркас**. Модули здесь — заглушки, никуда не подключены, реальных
> вызовов/ключей/тестов нет. Оживает после выбора сервиса реконструкции + ключа.

## Целевой флоу

```
Видео-проходка (смартфон, несколько проходов)
   │
   ├─ [Gemini, gemini.js]  QC съёмки ДО реконструкции  ── плохо → подсказки, переснять
   │                        (покрытие/свет/скорость/несколько проходов)
   ▼
[Реконструкция, reconstruct.js]  видео → .splat/.ply     (KIRI / Polycam / turnkey, на GPU сервиса)
   │  submitJob → опрос getStatus (~7–20 мин) → resultUrl
   ▼
Хостинг результата (.splat)
   │
   ├─ [Gemini]  авторазметка сцен + описание объявления   (опционально)
   ▼
[SplatViewer]  рендер в браузере/на телефоне  →  тур  →  Поделиться
```

## Разделение ответственности

| Слой | Кто | Делает | Статус |
|---|---|---|---|
| Реконструкция (видео→3D) | внешний сервис (GPU) | собственно Gaussian Splatting | нужен ключ |
| Умный слой | Gemini | QC съёмки, разметка, описания | нужен ключ, через бэкенд-прокси |
| Просмотр | наше приложение | `@mkkellogg/gaussian-splats-3d` рендерит .splat | **готово к подключению** |
| Хостинг/шаринг | наш бэкенд/Blob | хранит видео и .splat, ссылки | нужен бэкенд |

## Что осталось внешнего

1. **Ключ сервиса реконструкции** (KIRI рекомендуется).
2. **Ключ Gemini** — только если включаем умный слой; вызовы через бэкенд-прокси, не из браузера.
3. **Бэкенд/хранилище** для загрузки видео и раздачи .splat (для кросс-девайс шаринга).
